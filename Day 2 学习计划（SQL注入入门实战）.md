# 📚 理论学习（必须理解）

## 1️⃣ SQL注入是什么？

DVWA 中的 SQL 注入本质：

> 用户输入被拼接进 SQL 语句，导致数据库执行了“非预期命令”

---

## 2️⃣ 一个最典型的错误代码（核心）

```sql
SELECT * FROM users WHERE id = '$id';
```

如果你输入：

```text
1' OR '1'='1
```

变成：

```sql
SELECT * FROM users WHERE id = '1' OR '1'='1';
```

👉 结果：条件永远成立

---

## 3️⃣ SQL注入成立的3个条件

必须同时满足：

- 输入进入 SQL 查询
    
- 没有过滤/参数化
    
- 返回结果可见变化
    

---

# 🔥 五、这里“具体看什么变化”？

你要对比三种变化👇

---

## 🟡 ① 内容变多了（最明显）

### 正常：

```
只显示一条用户
```

### 注入后：

```
显示所有用户
```

👉 说明：  
✔ SQL被绕过了

---

## 🔴 ② 页面结构变化

比如：

- 表格变长
    
- 数据变多
    
- 登录状态变化
    

---

## 🟠 ③ 返回内容不同

比如：

- 正常：User ID = 1
    
- 注入：User ID = 1,2,3,4
    

---

# 🧠 六、最关键判断逻辑（一定要记住）

你不是在看“有没有报错”，而是在看：

> 🔥 输入改变 → 数据库结果是否改变

---

# 🧪 七、你现在应该形成的思维

每次测试都问自己：

### ✔ 这个输入有没有影响数据库？

如果：

|输入|结果|
|---|---|
|1|一条数据|
|1' OR '1'='1|全部数据|

👉 结论：

✔ SQL注入存在
