现在进入 OWASP 中非常重要的一类：

# SSRF（Server-Side Request Forgery）

中文：

> **服务器端请求伪造**

核心一句话：

> **攻击者控制服务器向指定目标发起请求。**

注意：

不是攻击者直接访问目标。

而是：

```text
攻击者
   |
   | 提供URL
   ↓
目标服务器
   |
   | 发起请求
   ↓
内部资源 / 外部资源
```

---

### 1. 图片加载


### 2. URL检测

例如：

输入：

```
https://xxx.com
```

系统：

检测：

- SSL信息

### 3. 文件导入

例如：

```

```

---

### 4. 在线转换

例如：

```
URL → PDF

```

---

这些功能都有共同点：

# 二、SSRF请求模型

理解这个非常重要。

## 普通请求

浏览器：

```text
用户浏览器
      |
      |
      ↓
服务器
```

---

## SSRF请求

攻击者：

```text
http://target.com/api/fetch?url=http://xxx.com
```

服务器收到：

```text
我要访问：

http://xxx.com
```

于是：

服务器：

```text
target.com服务器
          |
          |
          ↓
       xxx.com
```

---

区别：

访问主体变了。

原本：

```text
攻击者 IP
```

变成：

```text
服务器 IP
```

---

# 三、SSRF为什么危险？

因为服务器所在网络环境：

通常比外部用户权限高。

例如：

公网：

```
Internet
```

服务器：

```
Web服务器

192.168.1.10
        |
        |
        |
数据库
192.168.1.20

Redis
192.168.1.30
```

攻击者：

无法访问：

```
192.168.1.20
```

但是：

SSRF：

```text
攻击者
 |
 ↓
Web服务器
 |
 ↓
192.168.1.20
```

实现：

内网访问。

攻击链：


# 十二、SSRF漏洞报告模板

## 标题

```
服务器端请求伪造漏洞（SSRF）
导致内网资源访问
```

---

## 漏洞描述

```
目标系统某接口允许用户控制服务器请求地址，
未对目标URL进行有效限制，
攻击者可利用该功能访问服务器内部资源。
```

---

## 复现

请求：

```http
GET /api/fetch?url=http://127.0.0.1
```

返回：

```
Internal Service Page
```

---

## 影响

攻击者可能：

- 探测内网资产
    
- 访问内部管理系统
    
- 获取敏感信息
    
- 在云环境获取Metadata信息
    

---

## 修复建议

### 1. URL白名单

限制：

```
允许访问固定域名
```

---

### 2. 禁止内网地址

禁止：

```
127.0.0.1
localhost
私有IP
```

---

### 3. 网络隔离

限制服务器：

访问内部资源。
