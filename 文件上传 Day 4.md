# 🔥 文件上传 Day 4：真实上传漏洞挖掘思路（SRC / 渗透测试流程）

今天开始脱离 DVWA 单点实验，进入真实渗透测试思维。

前面你学习的是：

```text
DVWA Low
 ↓
发现无校验

DVWA Medium
 ↓
发现Content-Type校验缺陷

DVWA High
 ↓
理解文件内容检测
```

真实项目不会告诉你：

> “这里有一个文件上传漏洞，请测试”。

你的工作流程是：

```text
信息收集
 ↓
寻找上传功能
 ↓
分析上传逻辑
 ↓
测试限制
 ↓
验证风险
 ↓
输出报告
```

---

# 一、如何寻找上传点（信息收集阶段）

真实项目第一步：

不是直接上传文件。

而是：

## 找所有可能产生文件写入的位置

---

## 1. 常见上传功能

### 用户头像

例如：

```
/user/avatar/upload
```

特点：

- 用户可控
    
- 通常保存永久
    

---

### 附件上传

例如：

```
/article/upload
/message/attachment
/workorder/upload
```

特点：

危险等级较高。

---

### 简历上传

例如：

```
/resume/upload
```

---

### 图片编辑

例如：

```
/image/crop
/image/upload
```

---

### 导入功能

例如：

```
/import/excel
/upload/csv
```

不要只想到图片。

---

# 二、Burp寻找上传点

真实测试：

Burp是核心工具。

打开：

```
Proxy
 ↓
HTTP history
```

观察：

请求：

```http
POST /xxx/upload
```

---

重点搜索：

Burp：

```
Filter
```

输入：

```
multipart
```

或者：

```
filename=
```

---

因为文件上传一定包含：

```http
Content-Disposition:
form-data;
filename=
```

例如：

```http
Content-Disposition:
form-data;
name="file";
filename="a.jpg"
```

---

# 三、上传点分析流程

找到上传接口后：

不要马上测试危险文件。

先建立基线。

---

## Step 1：正常上传

上传：

```
test.jpg
```

记录：

返回：

```
upload success

记录：

### 1. 上传路径

/upload/2026/07/test.jpg

### 2. 文件名变化

```

### 3. 是否可访问

浏览器访问：

```
http://target.com/upload/test.jpg
```

# 四、黑名单绕过原理

很多开发采用：

## 禁止危险后缀

例如：

代码：

```php
if($ext=="php")
{
    die();
}
```

思想：

黑名单。

---

问题：

攻击者寻找：

没有被禁止的危险后缀。

---

例如：

服务器支持：

PHP解析：

可能存在：

```
.php
.php3
.php5
.phtml
```

---

开发：

禁止：

```
.php
```

但是：

忘记：

```
.phtml
```

---

形成：

```text
黑名单

禁止几个

↓

遗漏其他

↓

绕过
```

---

# 五、大小写绕过

例如：

过滤：

```text
.php
```

但是：

没有考虑：

```text
.PHP
```

或者：

```text
.PhP
```

---

原因：

Windows文件系统：

大小写不敏感。

但是：

部分解析环境：

可能产生差异。

---

开发错误：

```php
if(strstr($file,'.php'))
```

没有统一：

lowercase。

---

正确：

```php
strtolower($file)
```

之后判断。

---

# 六、特殊字符绕过思路

历史环境可能存在：

例如：

```
test.php.
```

Windows：

保存时：

可能自动去掉最后点。

---

或者：

空格：

```
test.php 
```

---

但是：

现代系统：

这些绕过越来越少。

重点是理解：

> 文件名处理不一致导致安全问题。

---

# 七、白名单绕过原理

白名单：

例如：

只允许：

```
jpg
png
gif
```

比黑名单安全。

但是：

实现错误仍然有问题。

---

## 错误1：只检查文件名

例如：

代码：

```php
if(extension=="jpg")
{
upload();
}
```

问题：

只看名字。

---

攻击：

```
shell.jpg
```

里面：

```php
<?php code ?>
```

---

如果上传目录可执行：

危险。

---

## 错误2：只检查Content-Type

例如：

```php
image/jpeg
```

可以伪造。

---

## 错误3：只检查文件头

例如：

```
FF D8 FF

<?php code ?>
```

图片头检测可能通过。

---

# 八、解析漏洞理解（重点）

很多新人误解：

上传成功 = 漏洞。

实际上：

危险链：

```
上传
 ↓
保存
 ↓
访问
 ↓
服务器解析
 ↓
代码执行
```

缺一不可。

---

例如：

上传：

```
test.php
```

保存：

```
/upload/test.php
```

访问：

```
http://target.com/upload/test.php
```

服务器：

Apache:

看到：

.php

执行。

---

但是：

如果：

Nginx配置：

```
uploads/
禁止PHP解析
```

那么：

上传成功：

也只是：

普通文件。

---

# 九、Web服务器解析问题

## Apache

常见：

根据后缀：

```
.php
```

解析。

---

## Nginx

通常：

通过：

```
location ~ \.php$
```

匹配。

---

错误配置：

例如：

```
location ~ .php
```

可能匹配：

```
test.php.jpg
```

---

这就是：

解析差异。

---

|特殊文件名|路径处理|

# 十一、Burp批量测试思路

真实项目不会一个一个试。

Burp：

## Intruder

流程：

```
抓上传请求

↓

Send to Intruder

↓

Payload位置

↓

替换filename
```

---

例如：

原：

```
filename="test.jpg"
```

设置：

```
filename="§test.jpg§"
```

Payload：

```
test.php
test.php5
test.phtml
test.PHP
test.php.jpg
```

---

观察：

响应差异：

例如：

```
200
Upload success
```

或者：

返回长度变化。

---

# 十二、漏洞报告编写

企业报告格式：

---

## 标题

```
任意文件上传漏洞
Unrestricted File Upload
```

---

## 风险等级

根据影响：

一般：

```
High
```

如果可执行：

```
Critical
```

---

## 漏洞描述

示例：

> 系统文件上传功能未对用户上传文件进行严格校验，攻击者可上传恶意脚本文件，并在服务器端解析执行，导致服务器被控制。

---

## 复现步骤

登录系统

访问：

```
头像上传页面
```

上传：

```
test.php
```

访问：

```
上传路径
```

服务器执行。

---

## 修复建议

### 文件类型白名单

限制：

```
jpg/png
```

---

### 服务端检测

不要相信：

```
filename
Content-Type
```

---

### 随机保存文件名

例如：

```
uuid.jpg
```

---

### 上传目录禁止脚本执行

---
