# 一、什么是文件上传漏洞？

简单理解：

> 网站允许用户上传文件，但是没有正确限制上传内容，导致攻击者上传危险文件。

---

正常业务：

例如：

头像上传：

用户：

```
avatar.jpg
```

服务器：

保存：

```
/upload/avatar.jpg
```

---

问题：

如果攻击者上传：

```
test.php
```

服务器：

保存：

```
/upload/test.php
```

并且：

Apache/Nginx：

会解析PHP。

那么：

访问：

```
http://target.com/upload/test.php
```

服务器执行代码。

---

形成：

```
上传文件

↓

服务器保存

↓

访问文件

↓

服务器解析

↓

代码执行
```

---

# 二、文件上传漏洞核心问题

不是：

> 能不能上传文件

而是：

> 上传的文件在哪里？服务器会不会执行？

---

例如：

上传：

```
test.php
```

结果：

## 情况1

服务器保存：

```
/upload/test.php
```

访问：

```
执行PHP
```

危险。

---

## 情况2

服务器改名：

```
/upload/a8sd92.jpg
```

并且：

禁止解析。

安全。

---

# 三、文件上传安全检查有哪些？

开发人员通常做：

---

## 1. 后缀检查

例如：

PHP：

```php
$file=$_FILES['file']['name'];

$ext=end(explode('.',$file));


if($ext!="jpg")
{
    exit;
}
```

意思：

只允许：

```
.jpg
```

---

问题：

攻击者可能绕过。

---

## 2. Content-Type检查

上传请求：

HTTP里面：

有：

```
Content-Type
```

例如：

图片：

```
Content-Type:
image/jpeg
```

PHP文件：

```
application/x-php
```

---

开发：

判断：

```php
if($_FILES['file']['type']=="image/jpeg")
{
允许
}
```

---

问题：

这个值：

客户端可以修改。

---

## 3. 文件内容检测

例如：

检查文件头：

JPEG：

```
FF D8 FF
```

PNG：

```
89 50 4E 47
```

---

更安全。

# 五、Low级别分析

打开源码：

路径：

```
DVWA/vulnerabilities/upload/source/low.php
```

代码类似：

```php
<?php

if(isset($_FILES['uploaded']))
{

$target_path =
DVWA_WEB_PAGE_TO_ROOT
."hackable/uploads/";

$target_path .= 
basename(
$_FILES['uploaded']['name']
);


move_uploaded_file(
$_FILES['uploaded']['tmp_name'],
$target_path
);

}

?>
```

---

分析：

重点：

```php
move_uploaded_file()
```

作用：

移动上传文件。

---

这里：

没有：

- 后缀检查
    
- 类型检查
    
- 内容检查
    

---

所以：

任何文件：

都可以上传。

# 六、先上传正常图片

准备：

```

```

---

这就是：

# 七、HTTP文件上传结构拆解（重点）

一个上传请求：

结构：

```
POST /upload HTTP/1.1

Host:
target.com

Content-Type:
multipart/form-data;
boundary=----xxx


------xxx

Content-Disposition:
form-data;
name="file";
filename="test.jpg"


Content-Type:
image/jpeg


文件内容


------xxx--
```

---

几个关键点：

---

## filename

文件名：

```
test.jpg
```

服务器可能检查这里。

---

## Content-Type

客户端声明：

```
image/jpeg
```

但是：

可以修改。

---

## 文件内容

真正的数据。

# 九、今天先理解绕过思想（不要急）

文件上传绕过本质：

开发检查：

```
攻击者输入
      |
      ↓
服务器检查
      |
      ↓
文件保存
```

攻击思路：

寻找：

```
检查点
```

和：

```
实际执行点
```

之间的差异。

---

例如：

检查：

```
filename=test.jpg
```

但是保存：

```
test.php
```

或者：

检查：

```
Content-Type=image/jpeg
```

但是内容：

不是图片。

# 十、企业真实防御

正确方案：

## 1. 白名单

不要：

黑名单：

```
禁止.php
禁止.exe
```

因为：

永远漏。

使用：

白名单：

```
只允许:

jpg
png
gif
```

---

## 2. 随机文件名

不要：

保存：

```
test.jpg
```

改：

```
8f92jd82.jpg
```

---

## 3. 上传目录禁止执行

Apache：

例如：

```
uploads/
```

禁止PHP解析。

---

## 4. 文件内容检测

检查：

Magic Number。

---

## 5. 权限隔离

上传目录：

不要：

可执行权限。
